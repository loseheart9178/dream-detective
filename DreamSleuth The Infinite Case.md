# 《造梦侦探：逻辑重构》产品开发方案

## 1. 产品定位与价值主张

### 1.1 产品定位
**AI驱动的个性化谋杀解谜游戏平台**

- **目标用户**：
  - 核心用户：谋杀之谜爱好者、剧本杀玩家（18-35岁）
  - 扩展用户：推理小说爱好者、桌游玩家、聚会社交人群
  - 创新用户：喜欢体验AI创作的科技爱好者

- **市场定位**：
  - 差异化：传统剧本杀需要购买固定剧本，本产品通过AI生成无限关卡
  - 价格优势：无需重复购买，一次订阅无限玩
  - 便捷性：随时随地即开即玩，无需准备

### 1.2 价值主张

| 维度 | 价值描述 |
|------|----------|
| **无限创意** | 用户输入任意关键词，AI生成独一无二的案件，永远玩不腻 |
| **个性化** | 支持用户自定义难度、场景、风格，满足不同偏好 |
| **沉浸式体验** | AI生成场景图片+文字剧情，视觉+文字双重沉浸 |
| **低门槛** | 无需组织者，无需多人组队，单人随时随地可玩 |
| **社交性** | 支持生成后分享给好友一起推理，支持多人协作 |

### 1.3 问题解决

传统剧本杀行业痛点：
- ❌ 剧本一次性，成本高
- ❌ 需要提前准备，组织门槛高
- ❌ 需要凑人数，受时间地点限制
- ❌ 内容固定，缺乏惊喜
- ❌ 创作周期长，新本产出慢

我们的解决方案：
- ✅ AI无限生成，边际成本为零
- ✅ 即开即玩，无需准备
- ✅ 支持单人游玩，也支持多人
- ✅ 每次都是全新案件
- ✅ 实时生成，秒开新关卡

### 2.2 各模块详细功能说明

#### 模块一：关卡生成器

**1. 关键词输入解析**
- 支持用户输入1-5个关键词
- AI自动理解关键词意图（场景/时代/主题）
- 冲突检测：关键词冲突时提示用户调整
- 智能扩展：简单关键词自动补充合理背景

**2. 案件框架生成**
- 案件类型：密室杀人/不在场证明/暴风雪山庄/交换杀人等
- 基本信息：时间、地点、被害人、死因
- 核心诡计：基于关键词设计独特诡计
- 难度调节：根据用户选择调整线索数量和迷惑性

**3. 嫌疑人生成**
- 人数：3-5人（可配置）
- 每个嫌疑人包含：
  - 基本信息（姓名、年龄、身份）
  - 动机（与死者的恩怨）
  - 不在场证明
  - 隐藏秘密
  - 谎言/矛盾点

**4. 线索系统生成**
- 分类：
  - 现场线索（物理证据）
  - 嫌疑人证词矛盾
  - 背景线索（关系网）
  - 致命证据（锁定真凶）
- 数量：根据难度动态调整
- 排布：合理分布，引导推理节奏

**5. 结局验证逻辑**
- 正确答案：真凶+作案手法+动机
- 验证规则：用户需要同时指认真凶并还原手法才算正确
- 分步提示：推理错误时给出梯度提示

**6. AI图像生成**
- 场景全景图：案发现场鸟瞰/概览
- 关键线索特写图：重要证据的可视化
- 嫌疑人肖像：可选功能

---

#### 模块二：推理交互系统

**1. 场景探索**
- 交互式地图/场景图
- 点击区域查看详细描述
- 未发现线索区域高亮提示
- 已收集线索归档

**2. 线索收集**
- 线索卡片收藏册
- 支持线索组合推理
- 标注已分析/未分析
- 备注功能：用户可自行添加笔记

**3. 嫌疑人询问**
- 对话树系统
- 不同问题得到不同回答
- 追问机制：发现矛盾后可追问
- 测谎提示：AI标记证词矛盾点（帮助用户）

**4. 逻辑拼图**
- 思维导图式推理整理
- 用户可自行连接线索和结论
- AI辅助整理：自动整理时间线
- 自动检测逻辑矛盾

**5. 结论验证**
- 用户提交：指认真凶+简述手法
- AI评分：根据完整度打分
- 分步给分：猜对真凶给一半分，完整还原给满分
- 提示机制：猜错时逐步给出提示

**6. 真相还原**
- 完整剧情复盘
- 时间线整理
- 每个线索的作用说明
- 诡计原理解释
- 分享关卡功能：生成分享卡

---

#### 模块三：用户系统

**1. 基础账号**
- 游客/登录两种模式
- 进度云同步
- 偏好保存

**2. 收藏与历史**
- 收藏喜欢的案件
- 历史记录自动保存
- 继续游戏：断点续玩

**3. 成就系统**
- 推理新手（完成第一个案件）
- 密室专家（破解10个密室案）
- 神速侦探（3步内破案）
- 关键词魔法师（用奇怪关键词生成案件）
- 分享达人（分享5次案件）

**4. 社交分享**
- 生成案件分享卡
- 好友合作推理
- 排行榜：破案速度/数量

### 3.3 交互设计原则

1. **轻量化**：每屏操作不超过3个主选项，避免选择困难
2. **即时反馈**：AI生成过程中显示进度动画
3. **梯度提示**：不直接给答案，逐步提示
4. **可回撤**：支持随时回到上一步，重新询问嫌疑人
5. **辅助整理**：AI自动帮用户整理时间线，降低记忆负担

## 4. 技术实现方案

### 4.1 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **前端** | React + TypeScript + Vite | 开发速度快，生态好，适合比赛项目 |
| **样式** | Tailwind CSS | 快速原型开发，美观简洁 |
| **状态管理** | React Context + useReducer | 足够用，无需引入额外复杂度 |
| **后端** | Node.js + Express | 全JS栈，开发效率高 |
| **数据库** | SQLite 开发 → PostgreSQL 生产 | 开发阶段轻量，方便迁移 |
| **AI大模型** | OpenAI GPT-4o / 通义千问 / 文心一言 | 强大的文本生成和理解能力 |
| **图像生成** | DALL-E 3 / Stable Diffusion XL / 通义万相 | 高质量图像生成API |
| **部署** | 前端：Vercel / Netlify<br>后端：Render / 阿里云函数计算<br>数据库：Supabase | 免费额度足够，方便演示，一键部署 |

### 4.3 Prompt工程设计

**核心Prompt模板（简化版）：**

```
你是一个专业的谋杀之谜剧本创作者。
用户给出的关键词是：{{KEYWORDS}}
难度：{{DIFFICULTY}}，嫌疑人数量：{{NUM_SUSPECTS}}

请按照以下要求生成一个完整的谋杀解谜案件：

1. 遵循 "不可能犯罪 → 嫌疑人各有动机 → 线索指向不同方向 → 真凶隐藏巧妙 → 核心诡计逻辑自洽" 的结构
2. 所有线索必须指向唯一正确解答，逻辑不能有矛盾
3. 嫌疑人中至少有2人有明显嫌疑，但真凶只有一个
4. 难度为{{DIFFICULTY}}时，安排{{CLUE_COUNT}}条线索，其中{{RED_HERRING}}条干扰线索

请以JSON格式输出，包含以下字段：
{
  "title": "案件标题",
  "description": "案件背景简述",
  "scene": "案发现场描述",
  "victim": {"name": "", "age": "", "description": "", "cause_of_death": ""},
  "suspects": [
    {
      "name": "",
      "age": "",
      "occupation": "",
      "relation_to_victim": "",
      "motive": "",
      "alibi": "",
      "lies": "嫌疑人所说的谎言/矛盾点",
      "secret": "嫌疑人隐藏的秘密"
    }
  ],
  "clues": [
    {
      "id": "唯一ID",
      "location": "发现位置",
      "description": "线索描述",
      "is_fatal": true/false, 是否是锁定真凶的关键证据
      "is_red_herring": true/false, 是否是干扰线索
      "related_to": "指向哪个嫌疑人"

    "killer_id": "真凶ID",
    "method": "作案手法详细描述",
    "motive": "真凶动机",
    "step_by_step": "时间线步骤",
    "logic_explanation": "如何通过线索推理出真凶的逻辑链"

确保输出是合法JSON，不要有多余字符。
```

### 4.5 API接口设计

**核心端点：**

```
POST /api/case/generate
  Body: { keywords, difficulty, numSuspects }
  Return: { caseId, status }  // 返回任务ID，轮询状态

GET /api/case/generate-status/:caseId
  Return: { status, progress, caseData? }
  // status: queued/generating/done/error

GET /api/case/:id
  Return: caseData (去掉solution，对用户隐藏答案)

POST /api/case/:id/submit
  Body: { killerId, explanation }
  Return: { correct, score, feedback, solution? }

GET /api/case/:id/solution
  Return: full solution (只有完成后才能看)

GET /api/user/progress/:caseId
  Return: 用户在该案件的进度

POST /api/user/progress/:caseId
  更新用户进度

GET /api/user/cases
  返回用户历史案件列表

POST /api/user/cases/:id/favorite
  收藏/取消收藏
```

### 4.6 错误处理与降级策略

1. **AI生成超时**：
   - 后端设置超时：90秒
   - 超时后提示用户重新生成
   - 自动重试1次

2. **图像生成失败**：
   - 降级：使用纯文字描述，不影响游戏
   - 后台异步重试，下次打开显示

3. **API配额用完**：
   - 展示友好提示
   - 开发比赛阶段：切换备用API密钥

4. **输出JSON解析失败**：
   - 自动让AI重新格式化输出
   - 重试最多2次，失败后提示用户

### 6.1 原创性创新性亮点

| 亮点 | 创新性说明 | 怎么展示给评委 |

| **无限生成机制** | 传统剧本杀是固定内容，本产品实现了"输入关键词→完整关卡"端到端AI生成，真正实现无限内容 | 演示时现场输入评委给出的任意关键词，现场生成，震撼效果 |
| **结构可控生成** | 不是单纯让GPT随便写，而是通过Prompt工程输出严格结构化JSON，保证游戏逻辑完整性和可交互性 | 展示系统架构图，说明Prompt设计思路 |

1. **预先生成多个优质DEMO案件**：

## 7. 潜在风险与应对方案

| 风险 | 影响程度 | 发生概率 | 应对方案 |
|------|----------|----------|----------|
| **AI生成逻辑不一致** | 高 | 中 | 1. Prompt中强制要求逻辑自洽<br>2. GPT-4o本身逻辑能力强，问题不大<br>3. 生成后让AI自检一遍逻辑<br>4. 开放用户反馈，收集差案例子优化Prompt |
| **生成速度慢** | 中 | 中 | 1. 前端做好看的loading动画<br>2. 提示用户生成需要1-2分钟，降低预期<br>3. 后端异步生成，支持轮询<br>4. 比赛演示用预生成备用 |
| **API费用超支** | 中 | 中 | 1. 提前估算：一次生成约0.1-0.2元，100次才20元<br>2. 开发阶段用小模型调试，最后用GPT-4o生成演示案例<br>3. 设置调用配额告警 |
| **API KEY quota耗尽** | 高 | 低 | 1. 准备多个备用API KEY<br>2. 支持配置多个AI服务商，一键切换<br>3. 预生成多个案件供演示使用 |
| **生成内容不符合内容安全** | 中 | 低 | 1. Prompt中要求：禁止血腥暴力色情，遵守法律法规<br>2. 后端增加内容审核环节<br>3. 不允许生成涉及敏感内容的关键词 |
| **部署后网络问题，API调用失败** | 高 | 低 | 1. 比赛时自备手机热点备用<br>2. 准备本地运行版本，万一公网崩了本地跑 |
| **图像生成质量不稳定** | 低 | 中 | 1.优化Prompt提示词，提升质量<br>2.图像是加分项不是必须项，失败了纯文字也能玩<br>3.可以选择不生成图像，节省时间和费用 |
| **选手现场紧张忘词** | 低 | 中 | 1.准备好演讲草稿PPT<br>2.多排练几次<br>3.分工：产品讲思路，技术讲实现，每个人讲自己熟悉的部分 |
